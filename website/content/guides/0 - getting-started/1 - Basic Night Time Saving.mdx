# Basic Night Time Saving

One of the easiest ways and most common use cases for the Downscaler is using it for night time saving.

A good time to start saving resources at night could be from 6:00 PM to 8:00 AM.

Depending on your use case you can apply the night time saving to either namespace or workload level.

## Namespace Level

To set this up for all workloads in a namespace you have to annotate the namespace with the wanted time span.

This can be done using a `kubectl` command or by editing the namespace's yaml manifest.

{/* prettier-ignore */}
{/* <!-- markdownlint-disable MD033 --> */}

<Tabs>
  <TabItem value="kubectl" label="kubectl" default>
    Adding the downtime with kubectl is a single command:

    ```bash
    kubectl annotate ns my-namespace downscaler/downtime="Mon-Sun 18:00-08:00 UTC"
    ```

  </TabItem>
  <TabItem value="manifest" label="Edit manifest">
    If you want to edit the annotations manually with a yaml manifest file
    or by running `kubectl edit ns my-namespace` you have to add the following:

    ```yaml title="my-namespace.yaml"
    apiVersion: v1
    kind: Namespace
    metadata:
      name: my-namespace
      # highlight-start
      annotations:
        downscaler/downtime: "Mon-Sun 18:00-08:00 UTC"
      # highlight-end
    ```

  </TabItem>
</Tabs>
{/* <!-- markdownlint-enable MD033 --> */}

Now all workloads in my-namespace will be scaled down every day from 6:00 PM to 8:00 AM.

## Workload Level

To set this up for individual workloads you have to set an annotation on each workload you want to scale down.

This can also be done by either using a `kubectl` command or by editing the namespace's yaml manifest.

For demo purposes we will showcase this with a deployment.

{/* prettier-ignore */}
{/* <!-- markdownlint-disable MD033 --> */}

<Tabs>
  <TabItem value="kubectl" label="kubectl" default>
    Adding the downtime with kubectl is a single command:

    ```bash
    kubectl annotate deployment my-deployment downscaler/downtime="Mon-Sun 18:00-08:00 UTC"
    ```

  </TabItem>
  <TabItem value="manifest" label="Edit manifest">
    If you want to edit the annotations manually with a yaml manifest file
    or by running `kubectl edit deployment my-deployment` you have to add the following:

    ```yaml title="my-deployment.yaml"
    apiVersion: apps/v1
    kind: Deployment
    metadata:
      name: my-deployment
      # highlight-start
      annotations:
        downscaler/downtime: "Mon-Sun 18:00-08:00 UTC"
      # highlight-end
    spec:
      replicas: 3
      selector:
        matchLabels:
          app: nginx
      template:
        metadata:
          labels:
            app: nginx
        spec:
          containers:
          - name: nginx
            image: nginx:1.14.2
            ports:
            - containerPort: 80
    ```

  </TabItem>
</Tabs>
{/* <!-- markdownlint-enable MD033 --> */}

Now our deployment my-deployment will be scaled down every day from 6:00 PM to 8:00 AM.
